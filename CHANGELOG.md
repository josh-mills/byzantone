# Changelog Format Guidelines

This changelog follows the [Keep a Changelog](https://keepachangelog.com/) format. When adding entries, please follow these guidelines:

## Format Structure

```markdown
## [Version] – YYYY-MM-DD
### Added
- New features

### Changed  
- Changes in existing functionality

### Fixed
- Bug fixes

### Deprecated
- Soon-to-be removed features

### Removed
- Removed features

### Security
- Security improvements
```

## Change Types

- **Added** - New features
- **Changed** - Changes in existing functionality  
- **Deprecated** - Soon-to-be removed features
- **Removed** - Removed features
- **Fixed** - Bug fixes
- **Security** - Security improvements

## Notes

- Use "Unreleased" for changes not yet released
- Include date in YYYY-MM-DD format for released versions
- Use clear, concise descriptions
- Group related changes under appropriate categories
- This section is not parsed by the changelog system
